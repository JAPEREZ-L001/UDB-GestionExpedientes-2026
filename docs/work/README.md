# work/ — Espacio libre para ensuciarse

Esta carpeta es tuya. Aquí puedes documentar sin presión: notas de progreso, borradores, decisiones, capturas de pantalla, ideas descartadas. Nada de lo que escribas aquí afecta la documentación oficial del proyecto.

---

## Convención de carpetas

Crea una carpeta con este formato:

```
YYYY-MM-DD_nombre-feature/
```

Ejemplos:
```
2026-03-19_frontend-registro-busqueda/
2026-03-20_backend-arbol-avl/
2026-03-21_fix-validacion-carnet/
```

Dentro de cada carpeta, crea un `notas.md` usando la plantilla en `docs/templates/notas-work.md`.

---

## ¿Qué va adentro?

- Notas de progreso del día
- Borrador de declaración de IA (antes de pasarla al PR)
- Capturas de pantalla o sketches de la UI
- Pruebas rápidas y resultados
- Preguntas pendientes o decisiones que tomaste

---

## ¿Qué NO va aquí?

- Código fuente (eso va en el repositorio del proyecto)
- Archivos binarios o ejecutables
- Información sensible (contraseñas, tokens)
- Nada que deba estar en el repo de código (`UDB-GestionExpedientes-2026`)

---

## ¿Se sube al repositorio de docs?

Sí, pero es opcional. Si quieres que el equipo vea tu proceso, súbelo. Si es demasiado borrador, no pasa nada. Lo importante es que uses la carpeta para pensar en voz alta.

> Ver `docs/examples/flujo-completo-ejemplo.md` para ver cómo se usa en la práctica.
